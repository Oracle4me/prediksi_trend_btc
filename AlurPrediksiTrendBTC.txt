Kesimpulan panjang:
Sistem mengambil data historis Bitcoin dari Yahoo Finance atau file CSV, kemudian melakukan feature engineering untuk membangun berbagai indikator teknikal dari data Open, High, Low, Close, dan Volume. 
Selanjutnya sistem membuat label target berdasarkan Future Return selama 14 hari ke depan, di mana return positif diberi label Bullish (1) dan return tidak positif diberi label Bearish (0). 
Dataset kemudian dibagi menjadi data training dan testing sebelum digunakan untuk melatih model Random Forest, XGBoost, MLP, Simple RNN, dan LSTM. 
Setiap model menghasilkan probabilitas terjadinya tren naik yang kemudian dikonversi menjadi label Bullish atau Bearish menggunakan nilai threshold. Kinerja model dievaluasi menggunakan metrik klasifikasi seperti Accuracy, Balanced Accuracy, Precision, Recall, F1 Score, dan Confusion Matrix. 
Seluruh hasil evaluasi, model, dan prediksi disimpan serta ditampilkan melalui dashboard untuk memudahkan analisis dan perbandingan performa antar model.

Kesimpulan singkat:
Data historis BTC dipakai untuk membuat fitur.
Label dibuat berdasarkan apakah BTC naik/turun setelah 14 hari.
Model dilatih untuk mengenali pola sebelum kenaikan/penurunan.
Output model adalah probabilitas naik dan label trend.
Evaluasi memakai metrik klasifikasi, bukan metrik harga.
